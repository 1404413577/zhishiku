# 功能盘点

## 1. 文档目的

本文档用于盘点当前知识库项目已有功能，并为 V2 重构确定处理策略。后续开发应以本表为边界，避免重构过程中继续无序增加功能。

处理策略说明：

- 保留：V2 必须继续支持。
- 重做：功能方向保留，但实现方式需要重构。
- 简化：V2 MVP 只保留基础能力，增强部分后置。
- 后置：不进入 V2 MVP，等核心稳定后再做。
- 删除：功能价值低或维护成本过高，不再迁移。
- 已重构：功能已经完成基础服务化或模块拆分，后续只做收尾优化。

优先级说明：

- P0：核心路径，必须优先完成。
- P1：重要功能，影响主要体验。
- P2：增强功能，可后置。
- P3：低优先级或锦上添花。

## 3. V2 MVP 必须完成

- AppLayout 基础布局。
- 文档树。
- 文档 CRUD。
- 文件夹 CRUD。
- 文档移动。
- 标签。
- 编辑器基础能力。
- Markdown 阅读。
- IndexedDB 存储。
- 搜索。
- 设置基础项。
- 数据导入导出。

## 4. V2 MVP 明确后置

- 本地 AI。
- 复杂首页图表。
- 标签图谱高级交互能力。
- WebDAV 冲突合并。
- SEO 自动化增强。
- Excalidraw 完整编辑体验。

## 5. 删除或清理项

- 搜索调试工具在生产代码中的入口。
- `.bak` 备份代码文件。
- 页面内重复 Markdown 渲染逻辑。
- Store 内直接重建搜索索引的逻辑。
- 页面内直接访问 WebDAV、IndexedDB、本地文件系统的逻辑。
