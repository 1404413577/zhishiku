# 服务接口设计

## 1. 文档目的

本文档定义 V2 重构中的核心 Service 接口。后续页面和 Store 应优先依赖这些接口，而不是直接访问存储、AI Provider、搜索索引或同步实现。

接口示例使用 TypeScript 表达，具体实现可逐步迁移。

当前落地状态：

- `documentService` 已落地，当前接口名称偏向兼容旧 Store，例如 `create`、`remove`、`import`、`export`；目标接口中的 `createDocument`、`batchRemove` 等命名可在后续页面迁移时统一。
- `markdownService` 已落地，阅读器、编辑预览和静态文档可逐步收敛到同一渲染入口。
- `searchService` 已落地，封装底层搜索引擎和增量索引能力。
- `relationService` 已落地，支持文档关系图、标签共现图和双向链接回退图。
- `mindMapService` 与 `mindMapStorageService` 已落地，分别承担 AI 大纲生成和本地导图会话持久化。
- `AiService` 仍以现有 `ai.js` 为主，后续需要继续拆 Provider、错误模型和会话服务。

## 2. DocumentService

负责文档和文件夹的核心业务。

```ts
export interface DocumentService {
  list(): Promise<KnowledgeNode[]>
  get(id: DocumentId): Promise<KnowledgeNode | null>
  createDocument(input: CreateDocumentInput): Promise<KnowledgeDocument>
  createFolder(input: CreateFolderInput): Promise<KnowledgeFolder>
  update(id: DocumentId, patch: UpdateDocumentPatch): Promise<KnowledgeNode>
  remove(id: DocumentId): Promise<void>
  batchRemove(ids: DocumentId[]): Promise<void>
  move(id: DocumentId, parentId: DocumentId | null): Promise<void>
  togglePin(id: DocumentId): Promise<KnowledgeNode>
  toggleFavorite(id: DocumentId): Promise<KnowledgeDocument>
  importDocuments(payload: string): Promise<ImportResult>
  exportDocuments(ids?: DocumentId[]): Promise<string>
}
```

输入模型：

```ts
export interface CreateDocumentInput {
  title: string
  content?: string
  parentId?: DocumentId | null
  tags?: string[]
}

export interface CreateFolderInput {
  title: string
  parentId?: DocumentId | null
}

export interface UpdateDocumentPatch {
  title?: string
  content?: string
  tags?: string[]
  summary?: string
  parentId?: DocumentId | null
  isPinned?: boolean
  isFavorited?: boolean
}
```


图谱数据：

```ts

  nodes: GraphNode[]

}

export interface GraphNode {
  id: string
  label: string
  type: 'document' | 'folder' | 'tag'
  value?: number
  meta?: Record<string, unknown>

export interface GraphEdge {
  id: string
  source: string
  target: string
  type: 'parent-child' | 'wiki-link' | 'tag-cooccurrence'
```


同步策略：

- 冲突处理：先检测，后人工确认，不自动覆盖。

## 12. MindMapService

负责思维导图业务。

当前实现采用三层拆分：

- `domain/mindmap`：节点模型、Markdown 解析、JSON 导入清理和导出规则。
- `composables/mindmap`：布局、节点操作、撤销重做、持久化调用、导入导出。
- `mindMapService` / `mindMapStorageService`：AI 生成大纲和本地会话保存。

```ts
export interface MindMapService {
  list(): Promise<MindMapDocument[]>
  get(id: string): Promise<MindMapDocument | null>
  create(title?: string): Promise<MindMapDocument>
  save(map: MindMapDocument): Promise<MindMapDocument>
  remove(id: string): Promise<void>
  importMarkdown(content: string): Promise<MindMapDocument>
  exportJson(id: string): Promise<Blob>
  exportPng(id: string): Promise<Blob>
}
```

## 15. 错误模型

服务层应抛出统一错误：

```ts
export class AppError extends Error {
  code: string
  cause?: unknown
  recoverable: boolean
}
```

常见错误码：

| code | 含义 |
| --- | --- |
| `DOCUMENT_NOT_FOUND` | 文档不存在 |
| `INVALID_PARENT` | 非法父节点 |
| `STORAGE_UNAVAILABLE` | 存储不可用 |
| `PERMISSION_REQUIRED` | 需要权限 |
| `AI_CONFIG_REQUIRED` | AI 配置缺失 |
| `AI_REQUEST_ABORTED` | AI 请求已取消 |
| `SYNC_FAILED` | 同步失败 |
| `IMPORT_INVALID_FORMAT` | 导入格式错误 |
