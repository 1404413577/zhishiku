# 知识库系统架构分析

## 📋 项目概述

**项目名称**: 知识库 (Zhishiku)  
**技术栈**: Vue 3 + Vite + Pinia + Element Plus + Tiptap  
**主要功能**: 文档编辑、AI 对话、思维导图、搜索、离线支持

## 🎯 核心功能模块

### 1️⃣ **文档管理模块** (documents.js Store)

**职责**: 全局文档状态管理

```
documents Store (Pinia)
├── State
│   ├── documents[]        # 所有文档列表
│   ├── currentDocument    # 当前编辑文档
│   ├── workspaceMode      # 'indexeddb' | 'local' 存储模式
│   ├── localDirHandle     # 本地文件系统句柄
│   └── searchResults[]    # 搜索结果
│
├── Getters
│   ├── allTags            # 所有标签集合
│   ├── filteredDocuments  # 根据标签/搜索过滤
│   ├── documentTree       # 树形结构 (支持排序/置顶)
│   ├── stats              # 文档统计数据
│   ├── getDailyActivityData  # 日活跃数据
│   └── getTagCooccurrenceData # 标签共现关系
│
└── Actions
    ├── createDocument()   # 创建新文档
    ├── saveDocument()     # 保存文档
    ├── deleteDocument()   # 删除文档
    ├── getDocument()      # 获取文档内容
    ├── search()           # 全文搜索
    ├── importFrom()       # 导入文档
    ├── exportTo()         # 导出文档
    └── syncWithWebDAV()   # WebDAV 同步
```

**数据流**:
```
用户编辑文档 → Editor.vue 
    ↓
发出 'save' 事件 → EditorToolbar.vue
    ↓
调用 saveDocument() → documents Store
    ↓
保存到 storage (indexeddb 或 LocalStorage)
    ↓
触发搜索索引更新 → search.js
```

**AI写作流程**:
```
点击"AI帮写"
    ↓
输入标题弹框 (填充当前标题)
    ↓
点击"开始生成" → 弹框关闭
    ↓

    ↓

    ↓
流式接收 → 实时更新编辑器内容
    ↓
点击"停止生成" (红色按钮旁显示) → abort()
    ↓
生成完成 → 自动保存
```

### 3️⃣ **AI 对话模块** (ChatView.vue)

**职责**: 多轮对话和会话管理

```
ChatView.vue
├── 左侧边栏
│   ├── 创建新对话
│   └── 会话列表 (sessionList)
│       ├── 会话信息 (标题/更新时间)
│       └── 删除按钮
│
├── 右侧主区域
│   ├── 头部工具栏
│   │   ├── 引擎选择器 (online/local/ollama)
│   │   ├── 模型选择器 (动态列表)
│   │   ├── 刷新模型列表按钮
│   │   └── 归档按钮 (保存为文档)
│   │
│   ├── 消息展示区
│   │   ├── User 消息 (文本显示)
│   │   ├── Assistant 消息 (Markdown 渲染)
│   │   └── 输入提示/错误信息
│   │
│   └── 底部输入区
│       ├── 多行输入框
│       ├── 发送按钮
│       └── 停止生成按钮 (生成中显示) ◄── 新增
│
├── 状态管理
│   ├── sessions[] (多个会话)
│   ├── activeSessionId (当前会话)
│   ├── messages[] (当前会话消息)
│   ├── isGenerating (生成中标记)
│   └── abortController (用于停止生成) ◄── 新增
│
└── 主要特性
    ├── 多引擎支持 (Online/Local/Ollama)
    ├── 会话管理 (创建/切换/删除)
    ├── 历史消息裁剪 (避免超出上下文)
    ├── 对话归档 (保存为 Markdown 文档)
    ├── 停止生成功能 (传递 AbortSignal)
    └── Markdown 消息渲染
```

**对话生成流程**:
```
用户输入消息 + 点击发送
    ↓
创建 AbortController
    ↓
添加 User 消息到会话
    ↓
准备历史消息 (MAX_HISTORY_TOKENS = 3000)
    ↓
根据引擎类型调用 AI 服务:
  - local: localAiService.chatCompletion()
  - online/ollama: AIService.chatCompletion()
    ↓
传递 signal: abortController.value.signal
    ↓
流式处理: 实时更新 UI + 消息内容
    ↓
stopGeneration() 方法:
  └─ abortController.value.abort()
     ↓
     处理 AbortError → 保存已生成内容
```

### 4️⃣ **AI 服务层** (ai.js)

**职责**: 统一的 AI 接口管理

```
AIService (单例)
├── getConfigs()          # 获取当前引擎配置
│   ├── online (OpenAI 兼容 API)
│   ├── local (WebLLM 本地运行)
│   └── ollama (本地 Ollama 服务)
│
├── listOllamaModels()    # 获取 Ollama 模型列表
│
├── chatCompletion()      # 核心聊天方法
│   ├── 参数:
│   │   ├── messages[]          # 消息列表
│   │   ├── onChunk(delta)      # 流式回调
│   │   ├── onProgress()        # 加载进度回调
│   │   └── options {
│   │       ├── model: 覆盖模型
│   │       └── signal: AbortSignal ◄── 中止支持
│   │       }
│   │
│   ├── 路由逻辑:
│   │   ├── local → localAiService
│   │   ├── ollama → _ollamaChatCompletion()
│   │   └── online → OpenAI 兼容 API
│   │
│   ├── 流式处理:
│   │   ├── 解析 SSE 流 / NDJSON
│   │   ├── 调用 onChunk 回调
│   │   └── 支持中止 (AbortSignal)
│   │
│   └── 错误处理:
│       ├── 超时处理 (DEFAULT_TIMEOUT_MS = 120s)
│       ├── AbortError 处理
│       └── 网络错误处理
│
└── polishText()          # 文本润色专用方法
```

**引擎调度逻辑**:
```
chatCompletion() 调用
    ↓
switch(settings.aiEngine) {
  case 'local':
    └─ localAiService.chatCompletion()
       └─ 使用 WebLLM 在浏览器中运行模型
  
  case 'ollama':
    └─ _ollamaChatCompletion()
       └─ Fetch 到本地 Ollama API
       └─ NDJSON 流式处理
  
  case 'online':
    └─ Fetch 到 OpenAI API 或兼容服务
       └─ SSE 流式处理
}
    ↓
if (options.signal?.aborted) {
  └─ 抛出 AbortError
}
```

### **组件间通信模式**

#### 1. **Props + Emit** (父子通信)
```vue
<!-- Editor.vue -->
<EditorToolbar
  :title="documentTitle"
  :aiWritingLoading="aiWritingLoading"
  @save="saveDocument"
  @aiWrite="handleAIWrite"
  @stopAIWrite="stopAIWrite"
/>
```

#### 2. **Provide/Inject** (深层组件通信)
```javascript
// 如果需要，可以通过 Provide/Inject 传递编辑器实例
provide('editor', editor.value)
```

#### 3. **事件总线** (全局事件)
```javascript
// App.vue 中监听 WebDAV 同步完成
if (import.meta.hot) {
  import.meta.hot.on('docs-changed', async (data) => {
    // 刷新预设文档
  })
}
```

#### 4. **Router 参数** (页面导航)
```javascript
// 编辑文档
router.push(`/editor/${doc.id}`)

// 查看文档
router.push(`/view/${documentId}`)
```

## 🚀 新增功能集成

### **AI 帮写停止生成** (Editor.vue)

```javascript
// 1. 定义状态
const aiAbortController = ref(null)

// 2. 在 handleAIWrite 中创建 AbortController
aiAbortController.value = new AbortController()

// 3. 传递给 AIService
await AIService.chatCompletion(
  messages,
  onChunk,
  null,
  { signal: aiAbortController.value.signal }
)

// 4. UI 中显示停止按钮
<el-button 
  v-if="aiWritingLoading" 
  type="danger" 
  @click="stopAIWrite"
>
  停止生成
</el-button>

// 5. 停止方法
const stopAIWrite = () => {
  aiAbortController.value?.abort()
}
```

// 与 Editor.vue 逻辑类似
const abortController = ref(null)

const stopGeneration = () => {
  abortController.value?.abort()
}

// 在 sendMessage 中
abortController.value = new AbortController()
await AIService.chatCompletion(
  history,
  onChunk,
  null,
  { signal: abortController.value.signal }
)

## 🎯 架构特点

### ✅ **优点**

1. **模块解耦** - 功能模块独立，易于维护扩展
2. **状态集中** - Pinia Store 统一管理，避免 prop drilling
3. **多引擎支持** - AI 服务透明切换
4. **离线友好** - 完整的离线编辑和存储支持
5. **流式处理** - 大量数据不阻塞 UI
6. **中止机制** - AbortController 支持长操作取消

### ⚠️ **优化空间**

1. **缓存策略** - 可以增加更多的缓存层
2. **错误恢复** - 网络中断时的自动重试机制
3. **性能监控** - 更详细的性能指标收集
4. **批量操作** - 大量文档导入/删除的优化
5. **实时协作** - 当前未支持多用户实时编辑
