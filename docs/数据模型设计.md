# 数据模型设计

## 1. 文档目的

本文档定义 V2 重构后的核心数据模型，作为存储层、服务层、页面和迁移逻辑的共同依据。

核心原则：

- Markdown 是文档内容的主格式。
- 文档和文件夹统一为知识节点。
- 存储来源通过 `source` 字段表达，不散落在页面逻辑中。
- 时间字段统一使用 ISO 字符串。
- 新模型需要兼容旧数据迁移。

当前落地状态：

- `src/domain/document/types.ts` 已定义 `KnowledgeDocument`、`KnowledgeFolder`、`KnowledgeNode` 和 `LegacyDocumentNode`。
- 当前实现仍保留 `LegacyDocumentNode` 作为页面和 Store 的兼容类型，目标模型会随着页面迁移逐步替换旧字段。
- `DocumentSource` 当前已落地 `indexeddb`、`local-fs`、`preset`；`dynamic`、附件和完整 Workspace/Settings 模型仍属于目标设计。
- `src/domain/mindmap` 已落地节点模型、Markdown 解析、JSON 清理和导出规则；完整 `MindMapDocument` 持久化模型后续再统一。

## 11. 旧数据迁移规则

当前旧数据大致字段：

```ts
{
  id?: string
  title: string
  content?: string
  tags?: string[]
  parentId?: string | null
  isFolder?: boolean
  isPinned?: boolean
  isFavorited?: boolean
  isPreset?: boolean
  isDynamic?: boolean
  originalPath?: string
  createdAt?: string
  updatedAt?: string
}
```

迁移规则：

- `isFolder === true` 转为 `type: 'folder'`。
- 其他数据转为 `type: 'document'`。
- 缺失 `id` 时生成新 id。
- 缺失 `content` 时使用空字符串。
- 缺失 `tags` 时使用空数组。
- 缺失 `createdAt` 和 `updatedAt` 时使用当前时间。
- `parentId` 统一转为 `string | null`。
- `isPreset` 为 true 时 `source.kind = 'preset'`。
- `isDynamic` 为 true 时 `source.kind = 'dynamic'`。
- 默认 `source.kind = 'indexeddb'`。

## 12. IndexedDB 存储设计

建议使用以下 store：

| Store | 内容 |
| --- | --- |
| `documents` | `KnowledgeNode` |
| `settings` | `AppSettings` |
| `attachments` | 附件元数据和 Blob |
| `chat_sessions` | AI 会话 |
| `mindmaps` | 思维导图 |
| `metadata` | 数据版本、迁移状态 |

建议元数据：

```ts
export interface StorageMetadata {
  schemaVersion: number
  migratedAt: string | null
}
```

## 13. 本地文件夹映射规则

- `.md` 文件映射为 `KnowledgeDocument`。
- 文件夹映射为 `KnowledgeFolder`。
- 文件路径作为稳定 id 的候选来源。
- 文件名作为标题。
- 文件内容作为 Markdown content。
- 修改文件名等同于重命名文档。
- 图片可存入同级资源目录，例如 `.assets/{documentName}/`。

## 14. 数据版本

V2 初始数据版本建议：

```ts
export const CURRENT_SCHEMA_VERSION = 2
```

迁移策略：

- 启动时检查 `metadata.schemaVersion`。
- 旧版本先备份，再迁移。
- 迁移失败不删除旧数据。
- 迁移完成后写入新版本号。
