# 知识库页面功能需求

## 1. 文档目的

本文档用于 V2 重构时定义每个页面应该实现的功能、交互、数据依赖和验收标准。后续开发应优先满足本文件中的页面级需求，再考虑增强功能。

页面设计原则：

- 页面只负责用户交互和展示，不直接访问底层存储。
- 页面通过 Store 或 Service 获取数据。
- 长任务必须有加载、成功、失败、取消或重试状态。
- 空状态、错误状态、移动端状态必须明确设计。
- 核心页面优先保证稳定，不为了炫技牺牲使用效率。

## 2. 页面总览

| 路由 | 页面 | 优先级 | 页面定位 |
| --- | --- | --- | --- |
| `/` | 首页 | P1 | 知识库概览、快速入口、数据统计 |
| `/editor/:id?` | 编辑器 | P0 | 文档创建、编辑、保存、AI 辅助 |
| `/view/:id` | 阅读页 | P0 | 文档阅读、双链跳转、目录导航 |
| `/search` | 搜索页 | P0 | 全文搜索、标签筛选、结果定位 |
| `/md-docs` | 静态文档页 | P2 | 浏览项目内置 Markdown 文档 |
| `/chat` | AI 对话页 | P1 | 多轮 AI 对话、会话管理、归档 |
| `/graph` | 关系图谱页 | P2 | 文档、双链、标签关系可视化 |
| `/mindmap` | 思维导图页 | P2 | 导图编辑、导入导出、AI 生成 |
| `/settings` | 设置页 | P0 | 个性化、存储、同步、AI 配置 |
| `/about` | 关于页 | P3 | 项目信息、版本说明、作者信息 |

## 3. 全局布局 AppLayout

### 页面定位

全局应用框架，提供导航、文档树、工作区状态、主题切换和页面承载区域。

### 必须实现

- 桌面端左侧文档树。
- 移动端抽屉式文档树。
- 顶部导航菜单。
- 当前路由高亮。
- 新建文档入口。
- 新建文件夹入口。
- 文档树搜索。
- 标签过滤。
- 文档树拖拽移动。
- 文档置顶、收藏、删除。
- 批量选择和批量删除。
- 预设文档分区。
- 用户文档分区。
- 工作区状态展示：IndexedDB、本地文件夹、权限待恢复、错误。
- 本地文件夹挂载和断开。
- 暗色/亮色主题切换。

### 交互状态

- 文档加载中。
- 文档为空。
- 搜索无结果。
- 本地文件夹权限失效。
- 删除确认。
- 批量删除确认。
- 拖拽移动失败回滚。

### 数据依赖

- `documentStore`
- `workspaceStore`
- `settingsStore`
- `documentService`
- `workspaceService`

### 验收标准

- 用户可以从任意页面通过侧边栏打开文档。
- 文件夹可展开、折叠、创建子文档。
- 移动端可以打开抽屉并选择文档。
- 不允许删除非空文件夹，或必须提供递归删除确认。
- 本地工作区权限失效时，用户能看到恢复入口。

## 15. 通用页面验收清单

所有页面都应满足：

- 刷新页面后可恢复必要状态。
- 数据加载失败时有错误提示。
- 空数据时有空状态。
- 长任务有 loading。
- 危险操作有确认。
- 移动端基本可用。

- 不在页面中重复实现领域规则。
